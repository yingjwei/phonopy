1. 概述

本程序针对 2D 材料元素替换 研究场景设计。 替换元素后声子谱不稳定是常见难题 ── 许多替换方案在几何上看似合理， 但实际 DFT 计算后却出现虚频。

本程序基于前期累计的 125+ 组 DFT+phonopy 计算结果构建了声子稳定性数据库， 覆盖 3 种母体结构、30+ 种元素替换。给定一个 POSCAR 和要替换的元素， 程序会：

  1. 自动匹配母体结构类型
  2. 结合声子稳定性数据 + 化学启发式评分
  3. 输出排名推荐及详细得分
核心理念：已经算过的材料告诉我们什么替换方案能稳定、什么不能。 与其每次都从零启发式猜测，不如从数据库中找到最可能有良好声子稳定性的方案。

2. 安装与依赖

环境要求

  • Python ≥ 3.8
  • pymatgen (≥ 2023.0)
  • numpy

安装步骤

# 1. 克隆仓库
git clone https://github.com/yingjwei/phonopy.git
cd phonopy

# 2. 安装依赖
pip install pymatgen numpy

# 3. 运行
python recommend.py POSCAR --element V
提示：推荐在 Linux 集群上使用（本机或登录节点均可）。 程序仅需 CPU，无需 GPU。

3. 基本用法

python recommend.py <POSCAR> --element <元素符号> [选项]

参数说明

参数说明必需
POSCARVASP 结构文件是
--element要替换的元素符号（如 V, Cl, S）是*
--site要替换的位点编号（1-based）是*
--candidates候选列表（逗号分隔或内置组名）否
--exclude排除的元素否
--ox-state目标氧化态否
--top显示前 N 个否
--json输出 JSON 文件否
--no-auto不使用数据库自动候选否
--weights自定义 5 个评分权重否

* --element 和 --site 二选一，必须指定一个。

6. 高级选项

自定义权重

5 个维度的权重可通过 --weights 调整：

# 默认: [经验, 电子, 半径, 电负, 氧化] = [0.40, 0.25, 0.15, 0.10, 0.10]
python recommend.py POSCAR --element V --weights 0.50 0.20 0.15 0.10 0.05

内置候选分组

组名元素
all全部常见元素
tm过渡金属
main主族元素
chalcogenO, S, Se, Te
pnictogenN, P, As, Sb, Bi
halogenF, Cl, Br, I
early-tm前过渡金属
late-tm后过渡金属

指定氧化态提高准确性

# 显式指定 V 为 +4 价
python recommend.py POSCAR --element V --ox-state 4
氧化态影响离子半径匹配度。如果不指定，程序会自动从结构中猜测（需 pymatgen 支持）， 猜测失败时使用元素常见氧化态的第一个值。

7. 评分标准

评分分为两大维度：经验评分（基于真实声子计算结果）和 化学评分（基于元素特性）。

7.1 经验评分（权重 0.40）

根据 125+ 组 DFT+phonopy 计算数据库：

情况得分
该替换在同类结构中声子谱稳定+1.0
仅 MSX 形式测试过（数据可靠性较低）+0.5
数据库无相关数据0.0（中性）
该替换在同类结构中声子谱不稳定-0.5

配对稳定性：某些金属的稳定性取决于共存的卤素（如 Ti 仅在 Cl 存在时稳定， 与 F 或 Br 搭配时不稳定）。此时程序会检查结构中的共存的卤素种类并相应调整得分。

7.2 电子构型（权重 0.25）

同族元素价电子数相同，d 电子构型最相似：

  • 同族（相同价电子数）：+0.50
  • 同区块（d→d, p→p）：+0.20
  • 邻族（d 电子数相近）：+0.15
  • 同行（轨道能级相近）：+0.10

7.3 离子半径（权重 0.15）

基于 Shannon 离子半径。半径差异过大会导致晶格畸变 → 声子虚频。 评分公式：exp(-2 × |Δr| / r_host)，差异越小分数越高。

7.4 电负性（权重 0.10）

Pauling 电负性差异越小 → 成键特性越接近 → 电子结构变化越小。

7.5 氧化态（权重 0.10）

与目标氧化态完全匹配 → 1.0；±1 内 → 0.8；±2 内 → 0.5。 氧化态不兼容会导致电荷失配和声子虚频。

9. 常见问题

Q: 程序算出的高分方案是否会 100% 声子稳定？

不能保证。 数据库提供的经验分只说明过去在类似结构中 该替换是稳定的。新材料可能有不同的具体结构参数（晶格常数、层数、磁性序等）， 最终稳定性仍需实际 DFT + phonopy 验证。

Q: 我的结构不在数据库中怎么办？

程序会自动降级到纯化学启发式评分（与 screen.py 相同）。 同时会给出"未知结构"提示。可以通过 --no-auto --candidates all 手动指定候选范围。

Q: 为什么有些元素同时出现在稳定和不稳定列表中？

这是因为配对稳定性效应。例如 Cr 在 V₄S₉Br₄ 型结构中，与 Cl 或 F 搭配时稳定， 但与 Br 搭配时不稳定。程序会自动检查结构中共存的元素来调整得分。

Q: 什么是 MSX-only 形式？

部分过渡金属仅在 MSX（M:S:X = 1:1:1）比例下测试过，而非完整的 M₄S₉X₄ 形式。 这些数据可靠性较低，得分会打折。

Q: 如何添加新的声子计算结果到数据库？

编辑 phonon_data.py，在对应结构的列表中增加或移除元素。 添加后立即生效，无需重新安装。

Q: 和 screen.py 有什么区别？

screen.py 是纯化学启发式筛选（零计算成本），recommend.py 在此基础上引入了真实声子计算结果作为经验评分。前者适合快速初筛，后者提供 数据驱动的精准推荐。
