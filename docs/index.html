元素替换 声子稳定性 筛选工具集

从启发式筛选到 VASP+phonopy 自动化流水线

2D Materials · Phonon · High-throughput

问题背景

在 VASP 计算中，通过元素替换调控材料性质是常见策略。但有些替换方案 无论怎么优化结构，声子谱总是出现虚频（imaginary modes）， 意味着该结构在动力学上不稳定。

本工具集提供三层次解决方案：

  • 方案 A — 零计算成本的启发式预筛选（本机运行）
  • 方案 B — 机器学习势快速声子筛选（需 GPU，预留框架）
  • 方案 C — 服务器端 VASP + phonopy 全自动流水线

方案 A：启发式筛选

零成本预筛选 本机运行

仅需 POSCAR 文件，几秒钟输出替换元素评分排名。支持交互模式、按配位环境分组推荐、分数氧化态自动计算。

五项评分维度

维度权重说明
电子构型 0.45 最重要。同族（相同价电子数/ d 电子数）最高分。 同区块（d↔d / p↔p）次之。电子排布决定了成键特性。
离子半径 0.20 Shannon 半径匹配度。太大/太小都会导致晶格畸变。
电负性 0.15 Pauling 标度差异。差异小 → 键性相似。
氧化态 0.10 候选元素是否存在目标氧化态。
周期邻近 0.10 同族 > 同行 > 邻族。
氧化态： 交互模式下会提示 氧化态 (直接回车=自动计算):， 输入数值直接使用，回车则自动计算。 自动规则：被替换元素若为 N、O、卤素、氧族、Pn族 取固定值（N=-3, O=-2, 卤素=-1, 氧族=-2, P/As=-3），其余通过电荷平衡推断。
氧化态评分：检查候选元素的经验价态表 （pymatgen common_oxidation_states）中有无接近目标价态的条目。 有则高分，不因排位靠后而扣分——排后面的也是合理价态。

交互模式详解

直接运行 python replace.py POSCAR 进入交互模式，自动依次执行：

  1. 显示结构摘要 — 文件路径、化学式、所含元素及原子数
  2. 列出原子坐标 — 带标签（N1, N2, V1, ...）和 Cartesian 坐标
  3. 按配位环境分组 — 显示每个元素的近邻配位环境、有效氧化态
  4. 等待输入 — 支持以下格式：
格式示例含义
元素符号 N 替换该元素的所有原子，各环境分别推荐 + 加权综合排名
符号+环境编号 N1 替换该元素的第 1 种配位环境下的全部 N 原子
（不是第 1 个 N 原子，是所有处于该环境的 N）

输入后询问氧化态：回车自动计算，输入数字则直接使用。

快速使用

# 交互模式：直接运行，选择要替换的原子
	python replace.py POSCAR

	# 一键推荐 V 的替代元素（按配位环境分组推荐）
	python replace.py POSCAR --element V

	# 一键推荐 Cl 的替代元素
	python replace.py POSCAR --element Cl

	# 指定候选列表
	python replace.py POSCAR --element V --candidates Nb,Ta,Mo,W,Cr

	# 指定氧化态（提高准确性）
	python replace.py POSCAR --element V --ox-state 4

	# 输出候选 POSCAR + JSON
	python replace.py POSCAR --element Cl --ox-state -1 	    --output-dir ./candidates --json results.json

方案 B：ML 势筛选

机器学习势快速声子计算 预留框架

使用通用 ML 势（MACE-MP-0 / CHGNet / M3GNet）替代 DFT 进行结构弛豫和声子计算， 速度提升 2-3 个数量级。适合大规模候选筛选。

所需环境

  • MACE 或 CHGNet（推荐 GPU，CPU 也可但较慢）
  • phonopy + ase
  • 典型耗时：数十秒/体系 vs. DFT 数小时
注意： 方案 B 为预留框架。如果将来配置了 ML 势环境， 可在 server_pipeline.py 中集成 phonopy + ML 势的声子计算模块。 当前建议使用方案 A + C 的完整 DFT 工作流。

方案 C：服务器流水线

VASP + phonopy 自动提交与分析 服务器端

从候选 POSCAR 自动生成完整的 VASP 弛豫 → phonopy 声子计算 PBS 作业， 并提供结果分析（含 Gamma 点虚频伪影识别）。

完整工作流

1. 本机启发式筛选
↓
2. 生成候选 POSCAR + 服务器作业
↓
3. 上传到服务器，提交 qsub
↓
4. phonon-setup 生成声子作业
↓
5. qsub 声子计算 → check band.yaml
↓
无虚频 → 稳定方案 ✓
有虚频 → 返回步骤 1 重新筛选

命令速查

# 1. 生成候选 POSCAR
python replace.py POSCAR --element V --candidates Nb,Ta \
    --ox-state 4 --output-dir ./candidates

# 2. 生成 VASP 弛豫作业（-r 指向含 POTCAR 的参考目录）
python server_pipeline.py setup ./candidates -r ./ref_opt -o ./vasp_jobs

# 3. 在服务器上提交弛豫
cd vasp_jobs/*/relax && qsub run_relax.pbs

# 4. 弛豫完成后，生成声子作业
python server_pipeline.py phonon-setup ./vasp_jobs --species "S V Nb"

# 5. 提交声子计算并分析结果
cd vasp_jobs/*/phonon && qsub run_phonon.pbs
python server_pipeline.py analyze ./vasp_jobs --json phonon_results.json

# 从单个 POSCAR 批量生成所有替换组合

Gamma 点伪影识别

分析工具通过 q 点位置（而非虚频大小）判断虚频性质：

  • 虚频仅出现在 Gamma 点 (q=0) → 声学求和规则 / 旋转求和规则伪影， 可通过 hiphive 修正，可安全忽略。无论虚频大小（-0.1 THz 还是 -6 THz）或数量多少，只要集中在 Gamma 点即视为伪影
  • 任何非 Gamma 点存在虚频 (q≠0) → 真实声子不稳定性， 无法通过 ASR 修正，该替换方案不可用
经验依据： 您报告的 VSF 体系在 Gamma 点出现 -6.39 THz 虚频（8 个 imaginary mode）， 但实验验证 VSF 稳定。通过 hiphive 的 enforce_rotational_sum_rules() 修正后虚频消失。 而 pho551 等体系在非 Gamma 点存在虚频，属于真实不稳定。

声子不稳定的常见原因

原因表现对策
尺寸失配 高频虚频、结构扭曲 用启发式筛选避免半径差异 >20%
电荷失配 虚频 + 电子不收敛 匹配氧化态；考虑额外掺杂补偿
费米面嵌套 特定 q 点虚频 (π,π) 考虑磁性、自旋极化、U 值
Jahn-Teller 局域结构畸变 检查对称性破缺可能性
声学求和规则 仅 Gamma 点小虚频 可忽略（自动识别）
